# 需求目标

taosgen 是时序数据产品的性能基准测试工具，功能强大但配置复杂。本需求旨在为 Claude Code 等 AI Agent 开发专用 Skills，实现：

1. **配置生成自动化**：根据用户提供的简单参数（如目标数据库类型、数据规模、测试场景等），自动生成符合 taosgen 规范的 YAML 配置文件
2. **运行命令辅助**：辅助用户构建和运行 taosgen 命令，支持本地和远程执行
3. **错误检测与修复**：自动检测配置文件中的常见错误并提供修复建议

目标用户：需要使用时序数据库性能测试工具的开发者、测试工程师、DevOps 人员。

# 功能需求

| 序号 | 功能类别 | 功能名称 | 功能描述 |
| --- | --- | --- | --- |
| 1 | 配置生成 | TDengine 写入配置生成 | 根据用户输入（连接信息、数据规模、表结构等）生成 TDengine 数据库写入测试的 YAML 配置 |
| 2 | 配置生成 | MQTT 发布配置生成 | 生成向 MQTT Broker 发布数据的 YAML 配置，支持动态主题 |
| 3 | 配置生成 | Kafka 生产配置生成 | 生成向 Kafka Broker 生产数据的 YAML 配置，支持多种序列化格式 |
| 4 | 配置生成 | CSV 数据源配置 | 支持从 CSV 文件读取数据生成配置 |
| 5 | 配置生成 | 数据生成器配置 | 支持随机数、顺序数、Lua 表达式等多种数据生成方式配置 |
| 6 | 配置验证 | YAML 格式验证 | 验证生成的 YAML 配置是否符合 taosgen 规范 |
| 7 | 配置验证 | 参数合理性检查 | 检查配置参数是否合理（如时间精度、批次大小等） |
| 8 | 运行辅助 | 本地运行命令生成 | 根据配置生成本地运行 taosgen 的命令 |
| 9 | 运行辅助 | 环境检查 | 检查运行环境是否满足要求（taosgen 是否安装、目标数据库是否可连接等） |
| 10 | 技能路由 | 统一入口 | 提供统一的 skill 入口，根据用户意图路由到 config 或 run 子技能 |

# 性能需求

1. 配置生成响应时间：简单配置（<100 行）应在 2 秒内完成
2. 配置验证时间：应在 1 秒内完成
3. 支持生成的最大配置规模：单文件不超过 1000 行 YAML

# 安全需求

1. **敏感信息处理**：
   - 数据库密码、Kafka SASL 密码等敏感信息不应硬编码在配置中
   - 支持使用环境变量引用敏感信息
   - Skill 不得索要或记录用户的实际密码

2. **命令安全**：
   - 生成的命令不应包含破坏性操作（如 `rm -rf`）
   - 涉及数据删除的配置（如 `drop_if_exists: true`）必须明确提示风险并要求确认

3. **配置审计**：
   - 生成的配置应包含版本信息和生成时间戳
   - 便于追溯配置来源

# 其他需求

## 兼容性需求

1. 支持 taosgen v0.3.0 及以上版本
2. 支持 Linux 和 macOS 系统
3. 生成的配置应兼容 TDengine 3.x 版本

## 接口需求

1. Skill 输入：支持自然语言描述和结构化参数两种输入方式
2. Skill 输出：
   - 配置文件路径（绝对路径）
   - 运行命令
   - 使用说明

## 易用性需求

1. 提供配置示例和模板选择
2. 对常见场景（智能电表、服务器监控等）提供预设配置
3. 错误信息清晰，包含修复建议
4. 支持配置增量修改（在现有配置基础上修改）

## 测试需求

1. 生成的配置必须通过 `cowork audit` 检查
2. 生成的配置必须通过 taosgen 的本地验证（`taosgen --dry-run` 或类似机制）
3. 支持生成测试数据量较小的配置用于快速验证
