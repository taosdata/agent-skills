# idmp-blog-generator

**版本**：0.2.0 · **作者**：Wang Zhe · **归属团队**：solution-center

根据 IDMP Demo 场景的全流程输出物，自动生成一篇结构完整、内容专业的行业场景博客。

---

## 目录结构

```
idmp-blog-generator/
├── SKILL.md              # 技能入口，定义输入规范与配图占位规则
├── README.md             # 本文件
└── references/
    └── 编写原则.md        # 博客写作的完整风格与结构规范（必读）
```

---

## 输入要求

| 输入项 | 是否必须 | 说明 |
|--------|----------|------|
| 行业场景名称 | 必须 | 如"地铁线路设备监控"、"污水处理"、"油井生产监控"等 |
| `outputs/industry_research.md` | 必须 | 由 `idmp-easyuse` 生成的行业调研报告，博客内容的主要数据来源 |
| 输出目录 | 必须 | 与 `outputs/industry_research.md` 路径一致 |

> 调研报告中的指标、告警规则、关键事件和面板需求均作为博客配图说明的数据来源，不得虚构数值。

---

## 输出

在与调研报告同级的目录下，生成一篇 Markdown 格式的博客文章，文件名格式建议为 `blog.md`。

---

## 博客结构（五章）

| 章节 | 主题 | 建议篇幅 |
|------|------|---------|
| 第一章 | 行业场景与关键矛盾 | 10%～15% |
| 第二章 | 业务问题：真正卡住运营的是什么 | 20%～25% |
| 第三章 | 解决思路：从数据到决策的闭环能力 | 20%～25% |
| 第四章 | 场景实践：业务流程如何被改变 | 25%～30% |
| 第五章 | 总结与展望 | 10% |

主线逻辑：**行业场景 → 业务问题 → 解决闭环 → 价值落地**
