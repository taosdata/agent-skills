# IDMP 行业场景博客生成器 (idmp-blog-generator)

根据 IDMP Demo 场景的全流程输出物，自动生成一篇结构完整、内容专业的行业场景博客。

**博客的核心主角是业务价值的转变，而不是 IDMP 的功能本身。** 读者关心的是"这能解决我的问题吗"，而不是"这个产品有哪些功能"。

## 输入要求

### 必须输入

- **行业场景名称**：如"地铁线路设备监控"、"油井生产监控"、"污水处理"等
- **编写原则**: 详见 `references/编写原则.md`，必须完全遵守其中的要求和禁止项。**生成每章标题（`##` 和 `###`）前，必须先查阅对应章节的"标题风格"说明（含三级标题通用规范），无对应规定的章节应保持行业语言、业务问题导向，禁止功能性句式。**
- **输出目录**：输出目录与`outputs/industry_research.md` 路径一致，博客内容需要参考调研报告中的行业背景和痛点分析部分。调研报告中的指标、告警规则、关键事件和面板需求均应作为配图说明的数据来源。

## 配图占位规范

博客正文中**必须**在关键论据处插入配图占位块。配图占位的作用是：让后续填图人员能够无歧义地理解"在哪里截图、截什么内容、要体现什么观点"。

### 配图数量要求

全文配图占位控制在 **5 张以内**，按以下优先级分配：

| 章节 | 配图数 | 选取原则 |
|------|--------|---------|
| 第一章（行业场景） | 1 | 资产层级全貌，建立场景认知 |
| 第二章（业务问题） | 1 | 选取最具代表性的 1 个痛点，不必每个问题各配 1 张 |
| 第三章（解决思路） | 1 | 能同时体现"业务对象统一 + 异常追因闭环"的联合视图 |
| 第四章（场景实践） | 2 | 1 张"新工作方式"核心场景视图 + 1 张关键事件详情或跨层级对比视图 |
| 第五章（总结展望） | 0 | 无强制配图，如有高价值趋势性视图可选配 1 张（计入总数） |

配图要求"少而精"：每张图必须直接证明正文的核心论点，删去所有"补充说明型"图。

### 配图占位格式

每个配图占位必须使用以下 Markdown 格式，放置在被支撑的正文段落**之后**：

```markdown
> 📷 **【配图占位】{图片标题}**
>
> _说明：{图片内容描述。包含：① 展示的具体指标/面板/事件名称；② 关键阈值或告警触发条件（引用调研报告中的具体数值）；③ 图片在此处用于证明哪个业务论点；④ 数据来源（对应调研报告中的哪个层级面板或事件）。}_
```

### 配图内容选取原则

1. **直接引用调研报告数据**：配图说明中的指标名称、告警阈值、事件条件必须与 `industry_research.md` 中的定义保持一致，不得虚构数值。

2. **优先选取以下类型的面板作为配图来源**：
   - 多指标联合趋势图（能体现跨维度关联）
   - 关键事件详情视图（能体现"从告警到追因"的闭环）
   - 跨设备/跨机组对比面板（能体现统一口径的价值）
   - 全厂/区域级 KPI 总览看板（面向管理者视角）
   - 带阈值标注的仪表盘或状态图（能直观体现合规边界）

3. **配图与正文强关联**：每张配图占位必须紧跟在其支撑的正文段落之后，说明中需明确"此图用于证明……"。

4. **不同层级都应覆盖**：配图应兼顾 L1 全厂、L2 机组/区域、L3 系统、L4 设备等不同层级，避免只集中在某一层级。

5. **关键事件优先呈现**：调研报告中定义的关键事件（`industry_research.md` 第五节）是高价值配图来源，应至少引用其中 1～2 个事件的详情视图。
