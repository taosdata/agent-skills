# IDMP 行业场景调研文档 (industry_research.md) 编写规范

本规范定义了 IDMP EasyUse 技能在执行“行业场景深度调研”步骤时生成的 `outputs/industry_research.md` 文档的结构与生成要点。

---

## 一、 文档基础信息 (Header)

*   **标题**：以场景中文名开头的调研报告标题。
*   **元数据**：必须包含以下三项：
    *   `生成时间`：YYYY-MM-DD。
    *   `场景名称`：由脚本生成的英文下划线 ID（如 `pvdf_production_line`）。
    *   `行业背景`：简洁描述该行业的特点、重要参数及生产逻辑（3-5句话）。

## 二、 层级结构 (I. Hierarchy Structure)

### 2.1 完整拓扑 (Complete Topology)
使用层级缩进的思维导图或目录树格式，展示从最高节点到最底层设备的引用关系。
*   **覆盖层级**：通常不少于 4 层（例如：L1 -> L2 -> L3 -> L4/设备）。
*   **场景示例**：
    *   **生产制造场景**：基地 -> 车间 -> 线体/区域 -> 设备
    *   **能耗监控场景**：物业/园区 -> 楼宇/楼层 -> 房间/配电室 -> 智能电表/温感
    *   **智慧园林场景**：管辖区 -> 公园 -> 灌溉片区 -> 传感器/控制器
*   **展示示例**：
    ```
    L1 某某园区
    ├── L2 1号楼
    │   ├── L3 中央空调系统
    │   │   ├── L4 末端控制阀-01
    │   │   └── L4 末端控制阀-02
    ```

### 2.2 层级说明表 (Level Description)
以表格形式解释每个层级的业务意义和节点示例：
| 层级 | 层级说明 | 节点示例 | 说明 |
| L1 | 项目/根节点 | XXX 基地 / 某某园区 | 顶面管控中心，主要展示整体核心 KPI |
| L2 | 分区/车间 | 聚合车间 / B栋大楼 | 主要单元，按物理或逻辑区域划分 |
| L3 | 线体/系统 | 引发剂配制区 / 配电监控系统 | 功能模块，按工艺或业务功能划分 |
| L4 | 设备/采集终端 | 轴承-A01 / 二楼智能插座 | 最小业务实体，数据产生的源头 |

## 三、 采集数据规范 (II. Collection Data Specification)

按**设备类型**分组定义所有采集数据量（采集物理量）。
*   **每类设备一个表格**。
*   **表格字段要求**：
    *   `物理量名称`：英文小写及下划线标识符。
    *   `中文标题`：必须为全中文，且语义明确。
    *   `数据类型`：数据库对应的类型（如 float, double, varchar, int, bool） 
    *   `采集频次`：明确单位（如 5s, 10s, 1min）。
    *   `单位`：遵循国际标准（如 °C, MPa, kg/h, kW, %）。如果没有单位，统一写 ` `。
    *   `最小值/最大值/典型值`：定义真实的数值分布范围。
*   **每类设备需严格按照格式输出**：
| 物理量名称 | 中文标题 | 采集频次 | 单位 | 最小值 | 最大值 | 典型值 |
| output_pressure | 输出压力 | 10s | MPa | 0 | 0.8 | 0.5 |

## 四、 各层级关键指标 (III. Key Indicators by Level)

定义后续用于"智能分析"聚合计算的指标。**必须参考 `references/idmp_analysis_abilities.md` 中描述的触发类型（滑动窗口、定时窗口、数据输入、状态窗口、事件窗口、会话窗口、计数窗口、异常检测）和计算能力（子元素聚合、窗口聚合等），合理设计每个指标。**
*   **分层列出**：必须从厂区到设备层逐层定义，覆盖所有中间节点（车间、区域）。
*   **表格字段要求**：
    *   `指标名称`：指标名称。
    *   `指标说明`：指标的说明。
    *   `计算公式`：指标计算的公式。如 `sum(${attributes['仪器功耗']})`。
    *   `触发方式`：触发方式。取值范围：Interval，Count，Event，Session。
    *   `时间间隔`：触发方式为Interval时，统计计算的时间间隔。如 10s，15m，1h。
*   **计算公式中属性引用规范（核心）**：
    *   公式中引用属性时，必须使用 `${attributes['属性名称']}` 的格式。例如： `${attributes['反应温度']}`。
    *   属性名称必须与采集数据规范中的中文标题完全一致。
*   **严禁跨设备计算**：同一个计算公式只能引用所在节点或其子节点的属性，且不得引用平级或其它路径的设备数据。
*   **严格按照格式输出**：
```
### L1 园区级 指标

| 指标名称 | 指标说明 | 计算公式 | 触发方式 | 时间间隔 |
| 园区总功耗 | 统计全园区所有设备每小时总功耗 | `sum(${attributes['仪器功耗']})` | Interval | 1h |

### L2 楼栋级 指标
**分析测试楼**

| 指标名称 | 指标说明 | 计算公式 | 触发方式 | 时间间隔 |
| 楼栋色谱总功耗 | 汇总色谱仪器每小时功耗 | `sum(${attributes['仪器功耗']})` | Interval | 1h |
```
## 五、 告警规则 (IV. Alarm Rules)

定义系统的自动预警逻辑。**必须参考 `references/idmp_analysis_abilities.md` 中描述的触发类型（如：数据输入触发做即时限值检查、事件窗口触发捕获持续性超限状态等）和计算能力（子元素聚合、窗口聚合等），合理设计每个告警。**
*   **分层列出**：必须从厂区到设备层逐层定义，覆盖所有中间节点（车间、区域）。
*   **表格字段要求**：
    *   `告警名称`：告警名称。
    *   `告警说明`：告警的说明。
    *   `触发条件`：触发告警的公式。如 `${attributes['罐内压力']} > 1.5`。
    *   `持续时长`：触发条件持续的时间。如 1s，10s，1m。
    *   `优先级`：告警优先级。取值范围：常规，警告，次要，主要，紧急。
*   **触发条件语法**：
    *   触发条件中引用属性时，必须使用 `${attributes['属性名称']}` 的格式定义布尔表达式，逻辑运算只能使用 and, or, not。例如：`${attributes['罐内压力']} > 1.5 and ${attributes['罐内压力']} < 2.0`。
    *   叶子节点告警的属性名称必须与采集数据规范中的中文标题完全一致。
    *   中间节点告警的属性名称必须与各层级关键指标中的指标名称完全一致。
*   **严禁跨设备计算**：同一个触发条件只能引用所在节点或其子节点的属性，且不得引用平级或其它路径的设备数据。
*   **严格按照格式输出**：
```
### L1 园区级 告警

| 告警名称 | 告警说明 | 触发条件 | 持续时长 | 优先级 |
| GC柱温超上限 | 柱温超过安全运行上限 | `${attributes['柱温']} > 330` | 60s | 主要 |

### L2 楼栋级 告警
**分析测试楼**

| 告警名称 | 告警说明 | 触发条件 | 持续时长 | 优先级 |
| GC柱温超上限 | 柱温超过安全运行上限 | `${attributes['柱温']} > 330` | 60s | 主要 |
```

## 六、 关键事件 (V. Key Events)

定义业务系统常用的关键事件，捕获并记录业务中具有重要意义的瞬时或持续性事件。**必须参考 `references/idmp_analysis_abilities.md` 中描述的触发类型（滑动窗口、定时窗口、数据输入、状态窗口、事件窗口、会话窗口、计数窗口、异常检测）和计算能力（子元素聚合、窗口聚合等），合理设计每个关键事件。**
*   **按需配置，无需每层级都设置**：选择最能体现业务价值的关键节点配置事件，不强制覆盖所有层级。
*   **表格字段要求**：
    *   `事件名称`：事件名称。
    *   `事件说明`：说明该事件的业务含义及重要性。
    *   `触发类型`：描述触发事件的分析触发类型（如：事件窗口、数据输入、滑动窗口等）。
    *   `计算条件`：该事件的计算条件表达式。
    *   `持续时间`：事件持续的时间，对持续时间没有要求的事件写“无”。
    *   `事件属性`：记录事件发生所对应的属性。
*   **严格按照格式输出**：
```
### L2 车间级 关键事件
**聚合车间**

| 事件名称 | 事件说明 | 触发类型 | 计算条件 | 持续时间 | 事件属性 |
| 批次改变 | 生产批次发生变化 | 状态窗口 | `${attributes['批次号']} | 无 | 批次变更 |
```

## 七、 可视化面板需求 (VI. Visual Panel Requirements)

规划展示看板，指导 `idmp-panel-creator` 自动化布局。**必须参考 `references/idmp_panel_types.md` 中描述的 15 种面板类型进行设计，合理选用趋势图、柱状图、饼图、仪表盘、条形仪表盘、表格、统计值、状态时间线图、状态历史图、资产列表、事件列表、散点图、事件趋势图等，确保面板类型多样化。**
*   **分层列出**：必须从厂区到设备层逐层定义，覆盖所有中间节点（车间、区域）和叶子节点。
*   **表格字段要求**：
    *   `面板名称`：面板名称。
    *   `面板说明`：面板说明。
    *   `图表类型`：图表使用的类型。必须包含多样化类型，**从 `references/idmp_panel_types.md` 中的15种类型中选取**。可用类型：趋势图、柱状图、饼图、仪表盘、条形仪表盘、散点图、表格、统计值、状态时间线图、状态历史图、资产列表、事件列表、事件趋势图。
    *   `展示指标`：图表中使用的属性名称。如：柱温。属性名称必须与采集数据规范中的中文标题完全一致。
*   **严禁跨设备计算**：同一个面板只能引用所在节点或其子节点的属性，且不得引用平级或其它路径的设备数据。
*   **严格按照格式输出**：
```
### L1 园区级 面板

| 面板名称 | 面板说明 | 图表类型 | 展示指标 |
| GC温度实时趋势 | 同时展示三个温区实时变化 | 趋势图 | 柱温、检测器温度、进样口温度 |

### L2 楼栋级 面板
**分析测试楼**
| 面板名称 | 面板说明 | 图表类型 | 展示指标 |
| GC温度实时趋势 | 同时展示三个温区实时变化 | 趋势图 | 柱温、检测器温度、进样口温度 |
```

## 九、 核心红线 (Critical Rules)

1.  **唯一基准原则**：此文档一旦生成，作为后续所有自动化生成的**唯一唯一指引导向**。后续生成逻辑不得自行编造属性名或逻辑。
2.  **中文名绑定原则**：所有 IDMP 内部逻辑绑定在“中文标题”上，因为 IDMP 前端和属性 ID 映射通常基于中文命名。
3.  **计算封闭性**：严禁出现如“设备 A 的温度 - 设备 B 的温度”这种跨设备节点的计算表达式。
4.  **数量保障（针对 Demo 生成类任务）**：
    *   设备类型数 ≥ 5 类。
    *   关键指标总数 ≥ 30 个。
    *   告警规则总数 ≥ 30 个。
    *   关键事件总数 ≥ 5 个（分布在 2 个以上节点）。
    *   面板需求总数 ≥ 30 个。
