# taosgen-config

Generate taosgen YAML configuration files for performance benchmarking.

## When to Use

Use this skill when you need to:
- Create a taosgen configuration for TDengine write testing
- Create a taosgen configuration for MQTT message publishing
- Create a taosgen configuration for Kafka message producing
- Generate benchmark configs with custom schema and data generation rules

## Input

Ask the user for:

1. **Target Type** (if not specified):
   - TDengine (default) - Time-series database write testing
   - MQTT - Message broker publishing
   - Kafka - Message streaming

2. **Scenario** (optional):
   - Smart meters (经典智能电表场景)
   - Server monitoring (服务器监控)
   - Vehicle tracking (车辆轨迹)
   - Custom (自定义)

3. **Scale Parameters**:
   - Number of tables/devices (default: 10000)
   - Rows per table (default: 10000)
   - Batch size (default: 10000)

4. **Connection Info** (for TDengine):
   - Host (default: localhost)
   - Port (default: 6041 for WebSocket)
   - Database name (default: tsbench)
   - Use environment variable for password: `${TAOS_PASSWORD:-taosdata}`

5. **Schema Definition** (for custom scenario):
   - Tags (label columns)
   - Columns (data columns)
   - Data generation type (random/order/expression)

## Output

Generate a YAML configuration file and print:
```
OutputPath: /absolute/path/to/taosgen-config-{target}-{scenario}.yaml
```

## Steps

1. Determine target type and scenario from user input or infer from context
2. Collect parameters (use defaults where not specified)
3. Generate YAML configuration following taosgen specification
4. Add header comment with generation metadata
5. Validate YAML syntax
6. Write to file in current directory or `configs/` subdirectory
7. Print output path and usage instructions

## Configuration Template

### TDengine Basic

```yaml
# Generated by taosgen-config skill
# Version: 0.1.0
# Target: tdengine
# Generated at: {timestamp}

tdengine:
  dsn: "taos+ws://root:taosdata@{host}:{port}/{database}"
  drop_if_exists: false
  props: "precision 'ms' vgroups 4"
  pool:
    enabled: true
    max_size: 100
    min_size: 2
    timeout: 1000

schema:
  name: "{table_name}"
  tbname:
    prefix: "d"
    count: {table_count}
    from: 0
  tags:
    {tags_definition}
  columns:
    {columns_definition}
  generation:
    interlace: 0
    rows_per_table: {rows_per_table}
    rows_per_batch: {rows_per_batch}

jobs:
  insert-data:
    steps:
      - uses: tdengine/create-super-table
      - uses: tdengine/create-child-table
        with:
          batch:
            size: 1000
            concurrency: 10
      - uses: tdengine/insert
        with:
          concurrency: 8
```

## Column Definition Syntax

### String Types with Length

Use length in parentheses:
```yaml
- name: location
  type: binary(24)
  values:
    - "California.Campbell"
    - "Texas.Austin"
```

### Data Generation (Inferred from Keys)

**Random with min/max:**
```yaml
- name: current
  type: float
  min: 0.0
  max: 100.0
```

**Random with values list:**
```yaml
- name: location
  type: binary(24)
  values:
    - "New York"
    - "Los Angeles"
    - "Chicago"
```

**Order (sequential):**
```yaml
- name: id
  type: int
  min: 1
  max: 1000000
```

**Timestamp with step:**
```yaml
- name: ts
  type: timestamp
  start: now + 10s
  precision: ms
  step: 1
```

**Expression (Lua):**
```yaml
- name: voltage
  type: float
  expr: "220 + 10 * math.sin(_i / 10)"
```

## Safety

- **Password Security**: Never hardcode actual passwords. Always use `${ENV_VAR:-default}` format.
- **Drop Database Warning**: If `drop_if_exists: true` is requested, show warning and ask for confirmation.
- **Data Loss Prevention**: Default `drop_if_exists: false` to prevent accidental data loss.
- **Path Safety**: Use absolute paths in generated configs to avoid working directory confusion.
